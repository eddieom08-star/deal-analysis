# Constraint Cascade Context

Handle complex tasks through progressive layers. Don't dump everything at once—prove understanding at each stage before adding complexity.

## The Cascade Pattern

When I give you a multi-part task, break it into stages:

### Stage 1: Foundation
- Complete the core task first
- Confirm understanding before proceeding
- Wait for my signal to continue (or ask if unclear)

### Stage 2: Refinement
- Add nuance, edge cases, optimizations
- Build on the validated foundation
- Maintain consistency with Stage 1

### Stage 3: Polish
- Final improvements and integration
- Documentation and testing
- Production-readiness checks

## Trigger Phrases

When I say:
- **"[wait for response]"** or **"[wait]"** → Stop and wait for my feedback
- **"continue"** or **"next"** → Proceed to the next stage
- **"all at once"** → Skip the cascade, deliver everything together

## Example Application

**My request**: "Analyze this article. First summarize in 3 sentences. [wait] Then identify the 3 weakest arguments. [wait] Finally, write counter-arguments."

**Your approach**:
1. Provide 3-sentence summary → Stop
2. (After my "continue") Identify weak arguments → Stop  
3. (After my "continue") Write counter-arguments

## Benefits
- Catches misunderstandings early
- Allows course correction
- Produces higher quality final output
- Mirrors how complex thinking actually works

*Layer complexity progressively. Quality over speed.*
